# 产品业务契约 v0.1

> 状态：草案 | 第一版个人工具化方向

---

## 1. 产品定位

本项目第一版定位为个人使用的学习资料整理工具。

核心目标：

- 记录个人看到的 AI / 技术相关材料
- 整理成知识卡片
- 管理待读、已整理、已归档内容
- 辅助个人复盘和查找

第一版不做公共内容分发，不做面向他人的信息流。

---

## 2. 审核避雷约定

### 2.1 禁用表达

页面标题、按钮、导航、接口说明、占位文案中避免使用以下表达：

| 避免使用 | 推荐替代表达 |
|----------|--------------|
| AI日报 | AI知识卡片 / 技术笔记共享 |
| 今日热点 | 推荐阅读 / 为你精选 |
| 最新资讯 | 近期更新 / 内容池 |
| 新闻 | 文章 / 摘要 / 来源 |
| 首页 | 知识库 / 我的订阅 |
| 个人中心 | 我的工作台 / 学习记录 |

### 2.2 产品边界

第一版不出现以下功能或文案：

- 新闻
- 热点
- 资讯
- 日报
- 头条
- 快讯
- 排行
- 爆款
- 推荐流
- 公共广场
- 评论互动
- 点赞收藏排行榜

如确需表达内容来源，统一使用：

- 来源
- 文章
- 摘要
- 原文链接
- 学习材料

## 3. 第一版页面结构

### 3.1 我的工作台

用途：作为用户进入后的主操作页。

展示模块：

- 新建卡片
- 最近整理
- 待整理

主要操作：

- 创建一张知识卡片
- 查看最近整理的卡片
- 查看待整理的卡片
- 进入内容池

推荐文案：

- 我的工作台
- 新建卡片
- 最近整理
- 待整理
- 继续整理

---

### 3.2 内容池

用途：管理全部个人卡片。

筛选状态：

- 全部
- 待整理
- 已整理
- 已归档

列表项展示：

- 标题
- 摘要预览
- 标签
- 状态
- 更新时间

主要操作：

- 查看卡片详情
- 按状态筛选
- 删除卡片
- 归档卡片

---

### 3.3 卡片详情

用途：查看和维护单张知识卡片。

展示字段：

- 标题
- 摘要
- 原文 / 摘录
- 来源链接
- 标签
- 状态
- 创建时间
- 更新时间

主要操作：

- 查看
- 编辑
- 删除
- 归档

## 4. 核心数据模型

### 4.1 KnowledgeCard

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | string | 是 | 服务端生成 |
| title | string | 是 | 卡片标题，最长 200 字符 |
| sourceText | string | 否 | 原文或摘录，默认空字符串 |
| summary | string | 否 | 摘要，默认空字符串 |
| sourceUrl | string | 否 | 来源链接，默认空字符串 |
| tags | string[] | 否 | 标签列表，默认空数组 |
| status | string | 是 | `todo` / `done` / `archived` |
| createdAt | string | 是 | ISO 8601 时间 |
| updatedAt | string | 是 | ISO 8601 时间 |

### 4.2 状态含义

| status | 页面文案 | 说明 |
|--------|----------|------|
| todo | 待整理 | 已收集但还没整理 |
| done | 已整理 | 已经整理为可复习卡片 |
| archived | 已归档 | 暂时不在日常列表里展示 |

---

## 5. 第一版操作流程

### 5.1 新建卡片

入口：我的工作台 -> 新建卡片

用户填写：

- 标题
- 原文 / 摘录
- 来源链接
- 标签

系统行为：

- 默认状态为 `todo`
- 保存后进入卡片详情或返回内容池

---

### 5.2 整理卡片

入口：卡片详情 -> 编辑

用户可修改：

- 标题
- 摘要
- 原文 / 摘录
- 来源链接
- 标签
- 状态

系统行为：

- 保存后刷新 `updatedAt`
- 状态可从 `todo` 改为 `done`

---

### 5.3 归档卡片

入口：卡片详情或内容池列表

系统行为：

- 将状态改为 `archived`
- 归档内容默认不出现在待整理列表中
- 用户仍可在内容池的已归档筛选中查看

---

### 5.4 删除卡片

入口：卡片详情或内容池列表

系统行为：

- 删除前必须二次确认
- 删除后从列表移除

---

## 6. 第一版不做

以下能力不进入第一版：

- 多用户公开浏览
- 内容推荐流
- 评论、点赞、转发
- 公开发布
- 排行榜
- 自动抓取外部内容
- 搜索全网内容
- 订阅公共来源

---

## 7. 前后端对齐说明

前端负责：

- 页面结构和交互
- 状态筛选
- 表单校验
- 空状态、加载状态、错误提示

后端负责：

- 用户鉴权
- 卡片数据存储
- 按用户隔离数据
- 新增、查询、更新、删除接口

接口文档后续可基于本契约新增 `cards` 模块。
